Personal Project

To Hex and Back

To Hex and Back is a tool that converts RGB color values into hexadecimal notation and vice versa.

The project uses regular expressions to match common ways of entering RGB and hexadecimal colors. Specifically, the hexadecimal input matches values either three or six characters in length, with an optional leading hash (#). The RGB input accepts three comma-separated numbers; the 'rgb' prefix and parentheses are optional.

When a valid value is entered into either input element, the converted value appears in the other field while also changing the background color of the page. Depending on the background color, the input fields and text either remain black or change to white so that maximum readability is maintained. This determination is the result of calculating a YIQ value from the RGB color. The YIQ equation returns a value between zero and 255. If the value is greater than or equal to 128 (which is halfway), the page elements turn black. If YIQ is less than 128, the inputs and text are changed to white.

The idea for To Hex and Back came about during basic JavaScript training while I worked as a junior front end developer at Hook Studios. The training entailed researching and defining commonly used methods on the Array, Number, String, and Function objects. As a believer in learning by doing, I wanted to further solidify what I had learned by immediately putting it into practice. As a result, To Hex and Back was born.

Technologies Used: HTML, CSS, JavaScript (ES6)
